## 1. Objetivo del Proyecto 1
Sistema ESP32 que:
1. Mide **temperatura** en tiempo real (¿damper/shock? probablemente multi-canal).
2. Controla **ventilación** (fans) para enfriamiento — on/off + velocidad.
3. **Geolocaliza** vía GPS (correlacionar datos térmicos con posición/ruta).
4. **GUI propia** (web) para visualizar señales + GPS y controlar en **local y remoto**.

## 2. Arquitectura propuesta (v0)
### Hardware
| Bloque | Componente (MVP) | Alternativa / upgrade |
|---|---|---|
| MCU | ESP32-WROOM-32 DevKit | ESP32-S3 (más RAM/PSRAM, USB nativo) |
| Temp | DS18B20 (1-Wire, sonda estanca, varios en 1 bus) | ⚠️ >110°C → termopar K + MAX31855 (SPI) |
| Fan | Fan 12V PWM 4-pin (25 kHz) + sense RPM | Blower + MOSFET IRLZ44N + flyback |
| GPS | u-blox NEO (UART2 16/17) @115200 — **funciona** (script del equipo) | — |
| Power | Banco: USB 5V | Vehículo: buck 12→5V + protección |
| GUI local | AP WiFi del propio ESP32 | — |

### Software / datos
- **Firmware**: arduino-cli + sketch `.ino` (Arduino-ESP32). Loop no bloqueante (millis); GPS se alimenta en cada iteración. Módulos: `gps.{h,cpp}` (+ temp/fan por venir).
- **Telemetría MVP**: ESP32 → HTTPS POST → Supabase (tabla `readings`). Downlink de control: ESP32 hace **polling** de tabla `commands`.
- **Fase 2**: MQTT (Mosquitto en VPS) bidireccional; n8n como bridge MQTT→Supabase + alertas.
- **Dashboard**: Next.js + Supabase Realtime en Vercel (PWA). GUI local servida por el ESP32 (WebSocket).
- **Modelo de datos (borrador)**:
  - `devices(id, name, last_seen)`
  - `readings(ts, device_id, temp_c[], fan_pwm, fan_rpm, lat, lon, speed, sats, hdop)`
  - `commands(ts, device_id, type, payload, ack)`
